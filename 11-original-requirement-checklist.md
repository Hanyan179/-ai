# 辅助推荐智能体原始需求清单

## 1. 需求定位

辅助推荐智能体适用于智能问答、资料检索等场景，综合用户身份、行为数据、上下文数据和长短期记忆，对用户进行精准画像，为用户提供个性化辅助推荐。

核心输出是个性化推荐列表，推荐内容包括但不限于：

- 政策文件。
- 通知公告。
- 业务指南。
- 数据报告。
- 服务事项。
- 待办任务。
- 协同事项。
- 工具入口。
- 模板资料。

## 2. 输入数据

### 2.1 用户行为数据

用于反映用户兴趣偏好和使用习惯。

- 浏览历史。
- 浏览内容标题。
- 浏览时间。
- 浏览时长。
- 浏览过的政策文件。
- 浏览过的通知公告。
- 浏览过的业务指南。
- 浏览过的数据报告。
- 操作记录。
- 服务事项申请类型。
- 服务事项申请频率。
- 待办任务处理情况。
- 协同事项处理情况。
- 搜索关键词。

### 2.2 上下文场景数据

用于识别用户当前处于什么环境和业务场景。

- 当前时间。
- 当前地点。
- 当前业务场景。
- 当前工作时间段。
- 当前会议场景。
- 当前办公地点。
- 当前页面场景。
- 当前页面对象。
- 当前页面推荐提示。
- 调用方可选上下文。

### 2.3 知识图谱数据

用于挖掘内容之间的关联关系和用户潜在需求。

- 政策文件与业务指南的关联。
- 服务事项与数据报告的关联。
- 待办任务与协同事项的关联。
- 文书模板与工具的关联。
- 政策文件与服务事项的关联。
- 用户行为与领域实体的关联。

### 2.4 长短期记忆数据

用于提升推荐连续性和个性化程度。

- 当前会话短期记忆。
- 最近浏览偏好。
- 最近推荐反馈。
- 长期点击偏好。
- 长期业务偏好。
- 常用推荐类型。
- 常用服务入口。

## 3. 编排逻辑

### 3.1 数据收集与整合

实时收集用户行为数据、上下文场景数据，并与知识图谱数据整合。

要求：

- 确保数据准确性。
- 确保数据完整性。
- 形成后续分析可用的统一上下文。

### 3.2 用户画像构建

基于用户行为数据构建用户画像。

画像维度包括：

- 兴趣爱好。
- 专业领域。
- 工作角色。
- 使用习惯。
- 常用业务。
- 常用推荐类型。

### 3.3 场景分析与感知

分析上下文场景数据，识别当前业务场景和用户环境。

要求：

- 结合用户画像判断当前场景下可能需求。
- 支持会议、办公地点、工作时间段、页面场景等因素。

### 3.4 潜在需求挖掘

结合用户画像和知识图谱挖掘潜在需求。

示例：

- 用户经常浏览某类政策文件时，推荐相关业务指南和数据报告。
- 用户操作年度总结创作时，推荐历史模板、数据工具和润色工具。

### 3.5 个性化推荐生成

根据用户画像、场景分析和潜在需求挖掘结果生成推荐。

推荐类型包括：

- 信息推荐：政策文件、通知公告、业务指南、数据报告。
- 服务推荐：服务事项、服务入口、申报指南。
- 任务推荐：待办任务、协同事项、任务提醒。
- 工具推荐：数据对接工具、内容优化工具、合规审查工具。
- 模板推荐：公文模板、年度总结模板、申报材料模板。

### 3.6 推荐结果评估与优化

通过用户反馈衡量推荐准确性和有效性。

反馈来源包括：

- 是否点击。
- 是否使用。
- 是否收藏。
- 是否忽略。
- 满意度评价。
- 后续操作转化。

## 4. 输出结果

### 4.1 个性化推荐列表

推荐列表应包含：

- 推荐内容标题。
- 推荐类型。
- 简要描述。
- 推荐理由。
- 相关链接。
- 来源。
- 建议操作。
- 优先级。

## 5. 辅助推荐角色管理

### 5.1 辅助推荐人格设定

身份定位：

```text
政务办公辅助推荐专员
```

人格特征：

- 精准。
- 高效。
- 贴心。
- 稳重。
- 合规。

能力要求：

- 数据整合能力。
- 精准匹配分析能力。
- 政策导向把控能力。
- 实时动态响应能力。

固定初始交互话术：

```text
您好，我是政务办公辅助推荐专员，可向您推送适配的政策文件、服务事项与待办任务。
```

### 5.2 角色切换及权限配置

要求：

- 支持不同推荐角色切换。
- 不同角色具备专属权限与推荐范围。
- 角色切换需结合办公场景。
- 推荐内容必须在权限范围内。

普通办公人员场景：

- 适用日常文件浏览、服务申请、任务处理。
- 可调用用户行为数据库、常规政策文件库、基础服务事项库。
- 推荐范围限于非涉密日常信息、普通服务、个人待办任务。
- 不可访问单位核心决策数据。

### 5.3 权限管控规则

要求：

- 明确不同用户、角色的推荐权限范围。
- 角色切换需要场景验证。
- 特殊角色需要审核条件。
- 避免越权推送。

示例：

- 切换紧急信息推送角色时，需要验证推送内容已通过单位办公室审核。

## 6. 辅助推荐对话流程管理

### 6.1 推荐意图识别

要求：

- 识别是否为推荐需求。
- 排除非推荐类需求。
- 对非推荐需求给出引导。

示例：

用户问：

```text
如何修改 OA 密码？
```

应识别为系统操作咨询，非推荐服务范畴，并引导至系统运维支持模块。

### 6.2 推荐场景模板库

要求内置高频推荐场景模板。

示例模板：

- 文书创作场景：模板推荐、工具推荐、数据支撑。
- 政策查询场景：政策解读、申报指南、配套服务。
- 年度总结创作场景：近 3 年历史模板、数据对接工具、自动润色工具。
- 政策申报场景：最新政策解读、申报材料模板、常见问题解答。

### 6.3 推荐超时与中断处理

推荐超时时：

- 提供基础推荐结果。
- 提示后续可生成精准推荐。

推荐中断时：

- 自动保存推荐进度。
- 重启后继续推荐。
- 避免用户重复输入。

### 6.4 推荐异常处理

异常类型包括：

- 行为数据缺失。
- 知识图谱关联失效。
- 检索失败。
- 用户输入错误。
- 系统模块故障。
- 输入乱码。
- 无意义字符。
- 非文字内容。

处理要求：

- 服务不中断。
- 标记异常原因。
- 提供替代推荐方案。
- 必要时引导用户重新输入。

### 6.5 多流程节点定义

推荐全流程节点包括：

```text
行为数据采集
→ 数据预处理
→ 知识图谱关联
→ 潜在需求挖掘
→ 推荐生成
→ 结果推送
→ 反馈优化
```

要求：

- 每个节点可标注进度。
- 能把控推荐节奏。
- 支持流程状态记录。

## 7. 辅助推荐提示词管理

### 7.1 推荐类型引导

要求：

- 引导用户明确推荐类型。
- 说明不同推荐适配场景。

推荐类型包括：

- 文书模板推荐。
- 工具推荐。
- 政策解读推荐。
- 服务事项推荐。
- 待办任务推荐。
- 通知公告推荐。

### 7.2 推荐特定场景指令

要求：

- 针对特定场景提供推荐指令。
- 使推荐结果与场景高度匹配。

示例：

- 年度总结创作场景：推荐近 3 年历史模板、数据对接工具、自动润色工具。
- 政策申报场景：推荐最新政策解读、申报材料模板、常见问题解答。

### 7.3 推荐上下文关联

要求：

- 关联用户前后操作。
- 保障推荐内容连贯。
- 避免需求遗漏。

示例：

- 用户使用年度总结模板后，后续推荐数据统计工具。
- 用户检索养老政策后，后续推荐养老服务合同模板。

## 8. 辅助推荐知识工程

### 8.1 推荐基础检索 NaiveRAG

针对明确需求，拆解关键词匹配知识库，快速返回推荐清单。

示例：

- 推荐本周待办任务。
- 拆解为待办任务、本周、部门。

### 8.2 推荐指令式检索 InstructRAG

面对模糊需求，先解析意图，明确推荐类型、核心场景、关联信息，再定向检索。

示例：

- 年度预算相关内容。
- 判定为信息、服务、任务综合推荐。

### 8.3 推荐结构化框架生成

将检索内容按结构组织。

推荐结构包括：

- 信息层。
- 服务层。
- 任务层。
- 工具层。
- 模板层。

### 8.4 推荐内容适配补全

结合用户偏好与场景规则补全推荐内容。

补全内容包括：

- 申领入口。
- 额度。
- 常用品类。
- 申报材料。
- 常见问题。
- 操作建议。

### 8.5 推荐合规性前置校验

推荐前校验：

- 涉密等级。
- 服务权限。
- 任务时效。
- 角色范围。
- 数据可见范围。

### 8.6 推荐结果排序

排序依据：

- 合规优先级。
- 需求匹配度。
- 用户兴趣偏好。
- 场景相关度。
- 任务紧急度。
- 内容时效性。

### 8.7 推荐个性化记忆

要求：

- 记录用户推荐偏好。
- 下次同类需求自动优先检索。
- 支持长期个性化优化。

### 8.8 推荐检索失败处理

要求：

- 提供替代推荐方案。
- 提示检索失败原因。
- 提供辅助入口或人工咨询入口。

## 10. 辅助推荐流程编排

### 10.1 推荐模糊需求与歧义处理

要求：

- 对用户行为文本和场景描述进行语义分析。
- 抽取关键槽位。
- 先通过历史行为和当前场景补全需求。
- 无法明确时触发引导式提问。

### 10.2 推荐步骤控制与异常恢复

要求：

- 对每个步骤进行状态监控。
- 支持暂停、继续、回退。
- 异常时先尝试规则化修复。
- 修复失败时跳转人工干预。
- 保留已完成数据，避免流程重启。

### 10.3 推荐规则流程图配置

要求：

- 提供可视化规则流程图。
- 预设高频推荐场景标准流程模板。
- 支持用户添加、删除节点。
- 支持设置条件分支。
- 节点关联明确规则。
- 设计完成后可生成可执行脚本。

## 11. MVP 分期标记

### 11.1 MVP 必做

- 开始节点四参数。
- 固定信息获取与上下文汇总。
- 推荐需求判定。
- 用户场景理解。
- 用户画像基础整理。
- 推荐任务规划。
- 基础检索。
- 候选内容标准化。
- 推荐排序。
- 推荐理由生成。
- 个性化推荐列表输出。
- 反馈记录占位。

### 11.2 MVP 简化实现

- 角色管理：固定为政务办公辅助推荐专员。
- 权限校验：默认接口返回均为用户可见数据。
- 知识图谱关联：先用规则或知识库关联替代。
- 长短期记忆：先读取已有上下文和历史行为，不做复杂学习。
- 场景模板库：先内置少量模板。
- 异常处理：先做基础降级回复。

### 11.3 后续增强

- 多角色切换。
- 完整权限配置。
- 用户偏好学习。
- 长期记忆更新。
- 知识图谱真实联动。
- NaiveRAG 与 InstructRAG 分层检索。
- 合规规则库。
- 超时与中断恢复。
- 推荐效果分析。
- 可视化规则流程图配置。
- Tools 工具体系封装。
