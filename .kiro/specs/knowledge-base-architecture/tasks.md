# 公文知识库内容规范

## Overview

本文档定义公文知识库的内容规范标准，是公司核心资产的建设指南。知识库的价值在于内容的规范性、完整性和可复用性。

---

## 一、知识库层级结构规范

### 1.1 四级层级标准

```
Level 1: 类目 (Category)
    │
Level 2: 文种/子类目 (Document Type)
    │
Level 3: 场景 (Scene)
    │
Level 4: 知识节点 (Knowledge Node)
```

### 1.2 命名规范

| 层级 | 命名格式 | 示例 |
|------|----------|------|
| Level 1 | 中文大类名 | 法定公文类型、格式规范库 |
| Level 2 | (序号) + 名称 | (一) 通知、(二) 报告 |
| Level 3 | 数字序号 + 名称 | 1. 会议类通知、2. 工作部署类通知 |
| Level 4 | (数字) + 具体名称 | (1) 会议召开方式常用表达 |

---

## 二、知识节点内容规范

### 2.1 节点必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| ID | 唯一标识，格式：类目-文种-场景-序号 | notice-meeting-召开方式-001 |
| 名称 | 节点名称 | 会议召开方式常用表达 |
| 路径 | 完整四级路径 | 法定公文类型 > 通知 > 会议类通知 > 常见套件 |
| 内容 | 具体知识内容 | [见下方内容格式] |
| 标签 | 分类标签 | 会议、通知、召开方式 |
| 来源 | 知识来源 | 《党政机关公文处理工作条例》 |
| 状态 | 完成状态 | 已完成/建设中/待建设 |

### 2.2 内容格式规范

每个知识节点的内容应包含：

```markdown
## [节点名称]

### 定义说明
[简要说明该知识点的用途和适用场景]

### 常用表达
1. [表达1]
2. [表达2]
3. [表达3]
...

### 使用示例
> [完整的使用示例]

### 注意事项
- [注意事项1]
- [注意事项2]

### 适用文种
- [文种1]
- [文种2]
```

## 四、知识录入流程

### 4.1 录入步骤

1. **确定路径**：明确知识所属的四级路径
2. **填写元数据**：ID、名称、标签、来源、状态
3. **编写内容**：按照对应类型的内容规范编写
4. **添加示例**：至少提供1个使用示例
5. **标注限制**：说明使用限制和注意事项
6. **关联节点**：关联相关的前置知识和相关知识
7. **审核发布**：经审核后发布

### 4.2 质量检查清单

- [ ] 路径完整（四级）
- [ ] 必填字段齐全
- [ ] 内容格式规范
- [ ] 有使用示例
- [ ] 来源可追溯
- [ ] 无敏感信息
- [ ] 标签准确
