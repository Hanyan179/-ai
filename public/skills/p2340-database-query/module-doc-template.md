# 模块信息文档模板

当获取到模块的完整信息后，按以下模板输出模块信息文档。

---

## 模板

```markdown
# 模块信息：{模块名称}

## 基本信息

| 属性 | 值 |
|------|-----|
| 模块ID | {module_id} |
| 模块名称 | {module_name} |
| 模块编号 | {module_code} |
| 模块类型 | {module_type} |
| 状态 | {status} |
| 业务数据表 | {db_table}（从元数据的 DB_TABLE 字段获取） |

## 表单

| 表单ID | 表单名称 | 类型 | 是否默认 | 状态 |
|--------|---------|------|---------|------|
| {form_id} | {form_name} | {form_type} | {is_default} | {status} |

## 列表

| 列表ID | 列表名称 | 类型 | 状态 |
|--------|---------|------|------|
| {list_id} | {list_name} | {list_type} | {status} |

## 元数据字段

| META_ID | 字段名称 | 类型 | 说明 |
|---------|---------|------|------|
| {meta_id} | {meta_name} | {meta_type} | 用户需求中的"{文字描述}"对应此字段 |

## 开发要素速查

- 打开表单URL中的 moduleId: `{module_id}`
- 打开表单URL中的 formId: `{default_form_id}`
- 列表JS文件对应的 listId: `{list_id}`
- JS中引用元数据字段: `data['{meta_id}']` 或 `data.{meta_id}`
```

---

## 使用说明

1. 每次获取模块信息后，按此模板生成文档
2. 元数据字段部分重点标注与用户需求相关的字段
3. 开发要素速查部分提供最常用的 ID，方便直接复制使用
4. 如果模块有子表单，在元数据字段部分增加子表单分组
