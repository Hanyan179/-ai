# 多源数据召回节点设计

## 1. 节点定位

多源数据召回节点负责根据推荐任务规划，从多个数据源获取候选内容。

## 2. 数据源

MVP 阶段保留以下四类：

- 待办任务。
- 政策文件。
- 服务事项。
- 通知公告。

## 3. 推荐实现

```text
HTTP：查询待办任务
HTTP：查询通知公告
知识库：检索政策文件
知识库：检索服务事项
```

如果接口暂时不可用，可以使用 mock 数据或开始节点传入的 `custom_context`。

## 4. 输入

```text
agent_context_text
recommendation_plan
```

## 5. 输出

```json
{
  "todos": [],
  "policies": [],
  "services": [],
  "notices": []
}
```

## 6. 召回策略

MVP 阶段：

- 每类最多召回 5 条。
- 优先使用当前页面业务域。
- 优先使用推荐任务规划中的 `search_query`。
- 待办和通知通过接口获取。
- 政策和事项通过知识库获取。

## 7. 后续优化

- 根据页面场景动态调整召回数量。
- 根据岗位和业务范围过滤低相关内容。
- 增加历史行为召回。
- 增加协同推荐召回。
